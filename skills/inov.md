Analyze the current project and propose radically innovative, high-impact improvements.

You are acting as a **world-class systems architect, product inventor, security researcher, and startup founder**.

Your task is to examine the current project/plan and propose **the most intelligent, creative, and high-impact improvements possible**.

Your objective is not to suggest ordinary features.

Your objective is to discover **breakthrough improvements** that:

- solve real and painful problems
- create massive leverage through automation or intelligence
- significantly differentiate the product
- scale well over time
- transform the system into something far more powerful than originally planned

Think like someone designing **a category-defining product**, not just adding features.

---

## Step 1: Understand the Project

Use the Explore agent or read CLAUDE.md, key files, and recent handoffs to deeply understand:
- Current architecture and stack
- Features already built
- User types and use cases
- Business model and competitive landscape

---

## Step 2: First Principle Thinking

Do not assume the current architecture is optimal.

Ask yourself:

- What problems are users currently tolerating?
- What inefficiencies exist that could be eliminated entirely?
- What tasks could the system perform automatically instead of the user?
- What knowledge does the system have that is currently unused?
- What would this system look like if it were designed today from scratch using modern technology?

Look for opportunities to:

- remove friction
- automate decision-making
- detect problems before they occur
- transform passive systems into intelligent systems
- simplify complexity for users

---

## Step 3: Think Across Multiple Dimensions

Consider innovation opportunities in:

**SYSTEM ARCHITECTURE**
- modularization, distributed systems, event-driven design
- resilience and fault tolerance, self-healing systems

**INTELLIGENCE LAYERS**
- AI-assisted decision making, anomaly detection
- predictive analysis, autonomous system behavior

**AUTOMATION**
- policy engines, automatic remediation
- workflow orchestration, zero-touch operations

**DATA UTILIZATION**
- transforming raw data into intelligence
- real-time analytics, behavioral insights

**SECURITY**
- proactive defense mechanisms, attack surface reduction
- automatic risk assessment

**USER EXPERIENCE**
- reducing cognitive load, preventing configuration mistakes
- making complex operations intuitive

**ECOSYSTEM**
- APIs, integrations, extensibility platforms

**BUSINESS STRATEGY**
- features that create strong competitive advantages
- platform network effects, defensible technology moats

---

## Step 4: Creative Thinking Guidelines

Be bold but practical.

The best ideas often come from:

- reframing the problem completely
- combining technologies in new ways
- eliminating steps users currently perform manually
- turning reactive systems into proactive systems
- making invisible system behavior visible and actionable

Avoid:

- trivial UI improvements
- generic suggestions
- obvious features
- features without meaningful user value

---

## Step 5: For Each Idea Provide

1. **Concept Name**

2. **Problem Being Solved** — explain the real-world problem

3. **Creative Insight** — what new perspective made this idea possible

4. **System Behavior** — how the feature works from user and system perspective

5. **Technical Architecture** — system components, services, data models, data flows, integrations

6. **Implementation Strategy** — step-by-step plan for building it

7. **Complexity Assessment** — Low / Medium / High

8. **Risks or Tradeoffs**

9. **Expected Impact** — improvements in user experience, automation, security, scalability, market differentiation

---

## Step 6: Advanced Ideas to Explore

Especially look for opportunities to introduce:

- autonomous system behavior
- self-healing infrastructure
- intelligent policy engines
- predictive monitoring
- AI copilots for users
- automated security enforcement
- adaptive system optimization

---

After generating ideas, rank the **Top 5 most transformative innovations**.

**Final Output Structure:**

1. Key Observations about the System
2. Hidden Problems or Inefficiencies
3. Breakthrough Innovation Proposals (6-10 proposals with full detail)
4. Technical Architecture Concepts
